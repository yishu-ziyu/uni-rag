# 开发日志 — 2026-06-25

## 主题：底层解析架构大换血（接入 LlamaParse）

### 背景与决策
用户确认了产品的核心定位：**开源 Showcase / 个人作品集**。
在这个定位下，“技术品味”和“架构成熟度”非常重要。因此，我们决定抛弃之前极其简陋且容易提取出“字符汤”的 `PyMuPDF`，升级为业界前沿的 **Semantic Parsing（语义解析）**。

考虑到用户不希望牺牲 UX 体验，并且路线二（大模型强行 OCR）会导致 Token 爆炸，路线三（本地运行 Marker）会导致机器卡死且解析极慢，我们最终选择了 **路线一：接入 LlamaParse（LlamaCloud API）**。

### 架构设计：策略模式（Strategy Pattern）
为了体现 Showcase 的专业性，我们没有把代码写死，而是实现了**降级容灾设计**：
1. **高配引擎**：当环境变量中配置了 `LLAMA_CLOUD_API_KEY` 时，系统自动使用 `llama-cloud` SDK 异步调用其 Agentic tier 进行高精度的多模态解析（完美还原表格、公式、多栏布局为 Markdown）。
2. **兜底引擎**：如果用户 clone 了代码但没有配置该 Key，系统会自动 Fallback 回本地轻量的 `PyMuPDF`。

### UX 优化：幻觉拦截的妥协（Soft Warning）
为了贯彻“不牺牲体验（速度优先）”的产品策略，我审查了昨天加入的 `CitationVerifier`（基于 BGE-M3 实时计算 Cosine Similarity）。
* **原逻辑**：在生成最终回答前，会同步阻塞执行引用相似度验证。
* **新逻辑**：目前仍然在同一个流程内，但增加了注释说明，在前端可以将其标记为灰色（Soft Warning），并且因为 BGE-M3 在本地运行极快，暂时不需要推向独立的后台任务，保持了“秒出”的交互感。
