# 前端交互调研报告：uni-rag 优化建议

> 2026-06-22 · 对标：ChatPDF、Humata、Perplexity AI、Dify、FastGPT

---

## 一、各产品关键观察

### ChatPDF
- 三栏布局：左侧导航（Chats/Folders/Tools）+ 中间 PDF 阅读器 + 右侧对话面板
- 上传后立即给 3 个建议问题按钮，降低首问门槛
- 底部 Fast/Quality 模式切换，让学生自主权衡速度与准确度
- 引用以页码链接形式嵌入回答，点击直接跳转到 PDF 对应页
- 侧边栏文件夹组织，支持按课程/主题分组

### Humata
- 极简 onboarding：登录后直接进入工作台，无明显教学弹窗
- 上传后左侧渲染文档，右侧自动生成 Summary + 3 个引导问题
- 经典"左文档、右对话"分栏布局
- 答案中直接标注数据来源页码/段落，点击跳回原文
- 四档定价清晰：Free / Student / Expert / Team

### Perplexity AI
- 首页只有一个搜索框，极简入口
- 答案旁嵌来源卡片，每个事实带可点击引用
- 结果页下方推荐追问建议，形成线程式研究流
- Focus 模式：学术/写作/财务等垂直场景过滤
- Pro Search / Deep Research：引导式深挖

### Dify
- 左侧边栏主导航，知识库以独立模块入口呈现
- 文档上传后展示 extraction/chunking/indexing 三段式进度
- 可视化节点编辑器（workflow canvas），检索配置以节点形式呈现
- Prompt IDE：左侧编辑 prompt，右侧实时对话测试
- RAG 引用以可溯源卡片形式展示（来源文档 + 片段）

### FastGPT
- 三栏结构：左侧应用列表 + 中间对话区 + 右侧配置面板
- 支持开场白配置、多轮上下文
- 回答附带引用来源标记，调试面板可查看完整调用链路
- 支持反馈引用并修改/删除
- 中国式 UX：微信/飞书扫码登录、国内模型优先

## 二、综合差距分析

| # | 差距 | 来源产品 | 当前 uni-rag 状态 |
|---|------|----------|------------------|
| 1 | **首问引导** | ChatPDF、Humata | 无。上传后直接空等用户输入 |
| 2 | **文档预览/缩略图** | ChatPDF、Humata | 只有文字列表，无视觉标识 |
| 3 | **回答质量模式切换** | ChatPDF | 无 Fast/Quality 选项 |
| 4 | **追问建议** | Perplexity、ChatPDF | 无 |
| 5 | **引用高亮精度** | Humata、SciSpace | 有引用 chips，但高亮是近似匹配 |
| 6 | **多文档摘要** | Humata | 无自动 Summary 生成 |
| 7 | **空状态设计** | 通用 | 有基础空状态，但偏功能性 |
| 8 | **KB 管理视觉化** | Dify、FastGPT | 只有下拉选择器，无独立管理页 |

## 三、Quick Wins（< 1 天）

1. **上传后给 3 个建议问题**
   - 效果：显著降低首次提问门槛
   - 做法：上传完成后在聊天区显示 3 个按钮（基于文档内容生成，或固定模板）

2. **文件列表加图标区分**
   - 效果：视觉上一眼区分 PDF / Markdown / URL
   - 做法：根据 `format` 字段显示不同 emoji 或图标

3. **引用 chips 加页码/段落提示**
   - 效果：用户知道引用来自文档哪个位置
   - 做法：在 citation chip 上显示 `[1] filename · p.3` 格式

4. **空状态文案优化**
   - 效果：更温暖，减少"工具感"
   - 做法：改为"先放一份材料进来，我会带着原文回答你的问题"

## 四、Medium-term（数天）

1. **追问建议生成**
   - 每次回答后，LLM 额外生成 2-3 个追问建议
   - 用户点击直接追问，形成对话流

2. **自动 Summary 卡片**
   - 上传完成后，LLM 生成一段 2-3 句摘要
   - 显示在文档列表旁，用户不点进去也能了解内容

3. **KB 管理页面**
   - 从下拉选择器升级为独立页面
   - 显示 KB 名称、文档数、创建时间、删除按钮
   - 参考 Dify 的 KB 列表卡片

4. **侧边栏文档缩略图**
   - PDF 显示第一页缩略图（用 WeasyPrint 或 pdf2image）
   - 非 PDF 显示类型图标 + 标题

## 六、建议下一步

从 P0 开始，先做"上传后建议问题"和"引用 chips 加位置信息"这两个改动，单次 commit 就能完成，用户能立刻感知到差异。
