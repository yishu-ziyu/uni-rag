# Product Specification: uni-rag

## 1. 目标与定位 (Vision)
**定位**：面向中文研究者的私有文档工作站。
**核心价值**：数据不出域的本地 RAG · 模型自由选择 · 国内直连可用。

uni-rag 解决的问题是：当用户需要基于敏感的商业合同、未发表论文、内部财报进行 AI 分析时，他们不敢/不能将其上传至 NotebookLM 或 ChatGPT。uni-rag 提供了一个在本地机器闭环处理数据、仅通过 API 访问大模型的方案。

## 2. 目标用户 (Target Audience)
- **学术研究者**：需要精读文献，需要严格的引用溯源。
- **产品/咨询/分析师**：处理不可外泄的研报、竞品数据、商业合同。
- **学生群体**：需要将教材转为闪卡、测验进行复习。

## 3. 核心用户旅程 (Golden Journeys)

### Journey 1: 本地文档的上传与问答
1. 用户在前端页面点击"上传第一份文档"。
2. 选择本地的 `.pdf` 或 `.md` 文件，系统在后台（不经过外网）进行文本提取、分块和向量化。
3. 文档出现在左侧边栏。用户点击选中。
4. 用户在底部聊天框输入问题，选择心仪的模型（MiniMax / StepFun）。
5. AI 返回答案，且答案下方附带带有原文档页码的可信引用块。

### Journey 2: 外部媒介的统一汇聚
1. 用户在左侧点击"添加链接"图标。
2. 粘贴 YouTube 视频链接或通用网页链接。
3. 后端拉取页面内容或视频字幕，清洗后存入本地向量数据库。
4. 链接作为一种"虚拟文档"出现在左侧列表，用户可以像对待 PDF 一样向视频内容提问。

### Journey 3: 知识加工 (模式转换)
1. 选中某篇已解析的文档。
2. 用户不仅可以选择"聊天"，还可以切换到：
   - **笔记**：自动抽取文档核心脉络。
   - **闪卡**：生成问答对用于间隔重复复习。
   - **测验**：生成带答案的单选题。
   - **知识图谱**：提取实体与关系，生成可视化 SVG 节点图。

## 4. 边界与不做的事 (Non-Goals)
- **不提供账户体系与云端同步**：为了保证 100% 数据不出域的叙事，当前版本不引入任何云端同步或账户注册机制，所有历史存于本地 localStorage 和本地 DB。
- **不自建推理模型**：核心在于"路由"而非"生成"，完全依赖外部高智商 API（如 MiniMax、StepFun、DeepSeek）或本地推理（Ollama）。
- **不做企业级权限管理**：这是一个单人生产力工具，不设计 RBAC 权限。
