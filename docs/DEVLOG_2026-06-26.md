# 开发日志 — 2026-06-26

## 主题：MinerU 接入 + PDF 解析降级容灾架构

### 背景

v1.0.0 上线后，产品定位明确为"开源 Showcase / 个人作品集"，"技术品味"和"架构成熟度"优先级高。但原方案选择的 LlamaParse 路线在实际使用中暴露两个问题：

1. **`llama-cloud` SDK 异步 API 与同步 Ingest Pipeline 不易桥接**——之前用 `asyncio` + `threading` 强行桥接，逻辑脆弱。
2. **LlamaCloud API key 在生产环境下不稳定**——之前测试期间频繁遇到鉴权失败，README 引导用户申请 key 体验门槛偏高。

调研后决定切换到 **MinerU（OpenDataLab 出品）**：
- 提供免登录的 Flash Extract 路线，零门槛。
- 提供精度更高的 Precision Extract 路线（需 token），国内访问稳定。
- CLI `mineru-open-api` 封装良好，可作为 SDK 替代。
- 对中文学术论文（公式 / 表格 / 多栏布局）解析质量优于 PyMuPDF。

### 架构设计：双引擎降级链（Fallback Chain）

```
parse_pdf(path)
    │
    ├─ if MinerU token 配置且 is_mineru_available():
    │      try: parse_file_via_api(path)         # MinerU v4 REST API
    │      except: warning log + 降级
    │
    └─ _parse_pdf_pymupdf(path)                  # 永远兜底
```

关键设计点：
- **MinerU 优先**：配置了 token 时优先使用云端高精度解析。
- **PyMuPDF 兜底**：网络错误 / API 异常 / token 过期 / 任何异常 → 静默降级到本地解析，**绝不阻断用户主流程**。
- **仅 warning 级别日志**：降级不视为错误，监控友好。

### v4 REST API 选型

MinerU 提供多套 API（v1 Agent / v3 / v4 precision / flash-extract）。最终选择 **v4 REST API（precision）**：

| 维度 | 决策 |
|------|------|
| 协议 | HTTPS REST + JSON（不用 SDK） |
| 端点 | `POST /api/v4/file-urls/batch` → 拿到 OSS 预签名 URL |
| 上传 | `PUT <oss-url>` 不带 Content-Type（OSS 签名不匹配） |
| 轮询 | `GET /api/v4/extract-results/batch/{batch_id}` 每 3s |
| 结果 | zip → `full.md` 提取 |
| Token | `Authorization: Bearer <JWT>` |

为什么不上 `mineru` SDK：测试发现 SDK 安装后并未暴露 `MinerU` 顶层类（文档与实际不符），CLI `mineru-open-api` 反而封装更完整，但调用外部进程开销大。直接 httpx 调 REST API 是最干净的路径。

### Embedder 内存问题修复

MinerU 提取的 Markdown 文本量显著高于 PyMuPDF（同一 PDF：46k chars vs 27k chars → 61 chunks vs 25 chunks）。一次 encode 全部触发 "Invalid buffer size: 33.97 GiB"。

**修复**：embedder 内部按 16 texts/batch 分批 encode。

```python
def embed(self, texts: list[str]) -> list[list[float]]:
    BATCH_SIZE = 16
    for i in range(0, len(texts), BATCH_SIZE):
        batch = texts[i:i + BATCH_SIZE]
        vecs = self.model.encode(batch, normalize_embeddings=True)
        results.extend(v.tolist() for v in vecs)
```

### 已知遗留风险

1. **当前 token 已过期**（JWT `exp` 字段为 2024-10）：CLI 和 REST API 均返回 401。fallback 到 PyMuPDF 工作正常，但 MinerU 路径暂时不可用。
   - 解决方案：用户去 https://mineru.net/apiManage/token 重新申请一个 token，更新 `.env` 即可。
2. **MinerU 返回页码丢失**：v4 precision API 返回扁平 Markdown，不带页码分段。如果产品要保留"引用显示第 X 页"功能，PDF 必须走 PyMuPDF 路径（或切回 v3 API）。
3. **OSS 上传 Content-Type**：预签名 URL 对 Content-Type 敏感，必须显式 `headers={"Content-Type": ""}` 关闭默认 `application/octet-stream`，否则签名 mismatch → 403。
4. **`config.py` 中残留 `llama_cloud_api_key` 字段**：预存在死代码，按规则不动。
